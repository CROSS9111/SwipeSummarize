# Release Draft Command

リリースの準備作業を自動化・効率化するためのコマンドです。
前回のリリース（タグ）からの変更点を集約し、リリースノート案を作成します。

## 実行方法

```bash
/release-draft [version]
```

- **version**: 今回リリースするバージョン番号 (例: `v1.2.0`)

## 実行手順

1. **変更点の収集**:
   - 最新のタグ（または指定されたベース）から現在までのコミットログを取得します。
   - マージされたプルリクエスト（PR）の情報を収集します。

2. **分類と整理**:
   - コミットメッセージやPRのラベル/タイトルに基づいて、変更内容を以下のカテゴリに分類します。
     - 🚀 新機能 (Features)
     - 🐛 バグ修正 (Bug Fixes)
     - ♻️ リファクタリング (Refactoring)
     - 📝 ドキュメント (Documentation)
     - 🔧 その他 (Others)

3. **リリースノート生成**:
   - 分類された内容をMarkdown形式のリリースノートとして整形します。
   - 重要な変更（Breaking Changes）がある場合は警告を目立つように配置します。

4. **リリース手順チェックリスト**:
   - `docs/operations/release/release_procedure.md` に基づく、リリース時に必要な作業リストを表示します。

## 使用例

```bash
/release-draft v1.5.0
```
