# 依頼

以下の【入力情報】を元に、GitHubのIssue用のコメントを作成してください。
出力は必ず指定の【マークダウン雛形】の形式に従ってください。

# 入力情報（ここを埋めてください）

- **コミットハッシュ**: [ 例: a1b2c3d ]
- **実施した作業/変更点**: [ 例: ログイン画面のレイアウト修正、バリデーション追加 ]
- **全体の進捗状況**: [ 例: UI実装フェーズの80%完了 ]
- **技術的決定・背景**: [ 例: ライブラリAはバグがあったためBに変更した。 ]
- **次の予定**: [ 例: テストコードの作成 ]

---

# 出力ルール

1. 文体は「です・ます」調で、チームメンバーへの報告として自然なトーンにする。
2. 「技術的決定・背景」は、後から振り返った時に意図がわかるように詳しく書く。
3. 変更がない項目は「特になし」とするか、行ごと削除して調整する。
