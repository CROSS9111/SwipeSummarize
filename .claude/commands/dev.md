# Unified Development Assistant (/dev)

`/dev` は、アクティブな開発フェーズにおける統合オーケストレーターです。
以下の専門コマンド群をあなたの代わりに呼び出し、複雑なタスクを遂行します。

## 実行方法

```bash
/dev [指示内容]
```

## 利用可能な機能（裏側で呼び出されるコマンド）

AIはユーザーの指示に基づき、以下のコマンドから最適なものを選択・組み合わせます。

| カテゴリ | コマンド | 説明 |
| :--- | :--- | :--- |
| **設計・実装** | `/spec` | **推奨**。統合設計書（design.md）を作成 |
| | `/spec-create` | 従来版。3ファイル構成の仕様策定 |
| | `/spec-execute` | 仕様に基づくコード実装 |
| **バグ修正** | `/bug-analyze` | エラーログ分析・原因特定 |
| | `/bug-fix` | 修正コードの生成 |
| **テスト** | `/test-generate` | 単体テストコードの生成 |
| **DB・API** | `/db-migration-helper` | DBマイグレーション生成・安全チェック |
| | `/api-sync` | Backend変更をFrontend型定義に同期 |
| **ドキュメント** | `/documentation` | 設計書をdocs/へ昇格 |
| | `/doc-update` | コード変更をドキュメントに反映 |
| | `/diagram-gen` | シーケンス図・クラス図・ER図の生成 |

## 実行プロセス

1.  **Plan**: 指示を分析し、実行すべきコマンドのリスト（タスクプラン）を提示します。
2.  **Execute**: ユーザーが承認したステップを順次実行します。

## 使用例

```bash
# 機能追加（仕様→実装→テスト→ドキュメント）
/dev ユーザー検索機能を追加して。仕様もテストもお願い。

# DB変更を含む実装
/dev Userテーブルにstatusカラムを追加して、APIも更新して。

# フロントエンド同期
/dev UserAPIを変えたので、フロントの型定義も直して。
```
